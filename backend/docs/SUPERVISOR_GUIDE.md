# Supervisor Command Center Operational Guide (Phase 8.2 Milestone 5)

## Overview

This guide details operational workflows for supervisors, Assistant Commissioners (ACP), and Deputy Commissioners (DCP) utilizing the NEXUS Assignment Governance system.

## Workflow Actions

### 1. Accepting a Recommendation
- Inspect top candidates ranked by M2/M3 scoring engines.
- Click **Accept Recommendation**. The system records the decision, creates a snapshot, and assigns the officer.

### 2. Overriding a Recommendation
- Select an alternative candidate.
- Select a **Standardized Override Reason** (e.g. `SPECIAL_EXPERTISE`, `LOCAL_KNOWLEDGE`).
- Provide **50+ characters of detailed justification**.
- System evaluates policy: if clear, assignment is executed immediately; if policy thresholds are met, request enters the **ACP/DCP Escalation Queue**.

### 3. Rejecting / Deferring
- **Reject**: Rejects proposed candidate list; case remains unassigned.
- **Defer**: Defers decision until specified timestamp or pending external intelligence/forensics.

### 4. Executive Approval (ACP / DCP Queue)
- Navigate to **Escalation Queue**.
- Review policy violation flags and supervisor justification.
- Click **Approve Escalation** to finalize executive assignment.
