## 2. Technical Enforcements
To guarantee this zero-variance behavior, Phase 7.0 implemented several strict safeguards:

1. **Ordinal Encoding Sorting:** In Crime Series feature extraction, categorical variables (like `Crime Category`) are mapped to integers. Python dictionaries do not guarantee order across runs. The engine uses `sorted(list(set(values)))` to guarantee that "Theft" always maps to `0` and "Assault" to `1`, regardless of OS or interpreter version.
2. **Cluster ID Hashing:** DBSCAN assigns arbitrary integer labels (`0`, `1`, `2`) to clusters based on memory traversal order. To make cluster IDs deterministic, the engine hashes the sorted list of FIR IDs contained within the cluster (e.g. `SHA1("FIR-1,FIR-2,FIR-3") -> "S-9f86d081884c7d659a2feaa0c55ad015a3bf4f1b"`). This guarantees the series ID never fluctuates.
3. **Cypher Order By:** All Neo4j queries end with `ORDER BY score DESC, n.id ASC` to ensure tie-breakers are handled deterministically, preventing UI flicker.

## Conclusion
The NEXUS Analytical Engine is strictly deterministic. A defense attorney or expert witness who downloads a snapshot of the database at time $T$ will be able to perfectly reproduce every intelligence inference and confidence score generated by the system.
